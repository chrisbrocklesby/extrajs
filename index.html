HTML extras

Extra power, minimum JS.

ExtraJS adds the missing extras to standard HTML: `((...))` bindings and inline behavior via `xjs` or `data-xjs`. Keep tiny interactions in the markup, and reach for `<script>` only when logic grows.

What ExtraJS adds to HTML

A small layer of extras that feel like they belong in the markup.

Inline JS with `xjs`

Write tiny interactions inline, right where the element lives.

HTML5-friendly `data-xjs`

Use `data-xjs` for standards-friendly attributes.

`((...))` bindings

Drop bindings into text or attributes and they update automatically.

Minimal state layer

`xstore`, `xcomputed`, `xwatch` only exist to support the HTML extras.

Inline JS, but on purpose

`xjs` is for the small, local stuff: toggles, simple state changes, quick timers, or wiring an event. When you need reusable logic, keep it in `<script>` and call it.

Clicking either button updates the live state and bindings above.

<button xjs="onclick = () => bump()">Inline click</button>
<button data-xjs="onclick = () => randomizeAccent()">Data attribute</button>

<div xjs="
  setInterval(() => {
    textContent = new Date().toLocaleTimeString();
  }, 1000);
">
  Time updates inline
</div>

ExtraJS keeps HTML first and JavaScript secondary. Use inline behavior for the small stuff, and keep larger logic in scripts.

Security note

Inline JS is powerful. Avoid untrusted content in `xjs`, and use CSP nonces or hashes if you need inline scripts in production.
