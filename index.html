Install extra.js

Use a local copy, or load it from a CDN. No tooling required.

Local file

Drop extra.js next to your HTML and include it:

INSTALL LOCAL 
<!DOCTYPE html>
<html>
<head>
  <script src="extra.js"></script>
</head>
<body>
  <h1>Hello (​(name)​)</h1>
  <script>
    x.store.name = "extra.js"
  </script>
</body>
</html>

CDN

Pull directly from jsDelivr (GitHub version):

INSTALL CDN 
<script src="https://cdn.jsdelivr.net/gh/chrisbrocklesby/extrajs@latest/extra.js"></script>

The globals are available as x and extra.

API summary

Most apps only need a handful of primitives. Everything lives on x (and the extra alias).

Store & reactivity

  • x.store â€“ reactive global state (proxied, saved in sessionStorage).
  • x.watch(path, fn) â€“ run fn(newVal, oldVal) when a path changes.
  • x.computed(name, fn) â€“ define derived state that tracks dependencies.

Templating & directives

  • (​(path)​) â€“ render store or computed values in text/attributes.
  • x-js="code" â€“ run inline JS when an element appears.
  • x-if / x-else â€“ conditional insertion.
  • x-show â€“ toggle hidden without removing the element.
  • x-bind:attr â€“ bind a property/attribute to a store path.
  • x-on:event â€“ attach event handlers with full JS (sync/async).
  • x-for="item in items" â€“ simple loops with local variables.

Config

extra.js exposes a reserved configuration object at x.config so future options can live in one place. At the moment there are no official flags â€” it is safe to treat it as a simple namespaced object for your own settings.

Example (user-defined only):

CONFIG 
// Reserved for future built-in options
// You can store your own app-level flags here.
x.config.appName = "My extra.js demo";
x.config.debug = true;

When official config options are added, they will appear under x.config and be documented in the README.
