Quick start

Three steps: include the script, set some state, and bind it in your HTML.

Step 1 ¬∑ Load extra.js

Use from a CDN or install locally. No tooling required.

CDN 
<script src="https://cdn.jsdelivr.net/gh/chrisbrocklesby/extrajs@latest/extra.js"></script>

Step 2 ¬∑ Define some state

Use x.store for global reactive state. It‚Äôs just a plain object that survives refreshes in sessionStorage.

STATE 
<script>
  x.store.name = "extra.js"
  x.store.count = 0
</script>

Step 3 ¬∑ Bind in HTML

Use (​(path)​) to read from the store and simple x-on handlers to update it.

HELLO WORLD 
<h1>Hello (​(name)​)</h1>
<p>Clicks: (​(count)​)</p>

<button x-on:click="x.store.count++">
  Click me
</button>

That‚Äôs enough to build simple dashboards, admin panels, marketing pages with reactive ‚Äúislands‚Äù, and more.

What extra.js can do

You can stop here and already be productive. When you need more, these are the main building blocks.

Core concepts

  • x.store ‚Äì reactive global state.
  • (​(path)​) ‚Äì text & attribute bindings.
  • x-js ‚Äì run inline JS on mount.
  • x.computed ‚Äì derived values.
  • x.watch ‚Äì listen to changes.
  • State persisted via sessionStorage.

HTML directives

  • x-if / x-else ‚Äì conditional blocks.
  • x-show ‚Äì show/hide with hidden.
  • x-bind:attr ‚Äì bind attributes & props.
  • x-on:event ‚Äì click/input/etc handlers.
  • x-for ‚Äì simple loops.

AJAX (x-http)

  • x-http ‚Äì send a request & swap HTML.
  • x-method ‚Äì override GET/POST/etc.
  • x-target / x-target-error ‚Äì where to put success / error.
  • x-swap ‚Äì inner, append, prepend, before, after, outer.
  • x-trigger ‚Äì when to fire (click, submit, load, every:ms‚Ä¶).
  • x-indicator ‚Äì show spinners while loading.
  • x-headers, x-form, x-json, x-confirm ‚Äì control headers, body and confirmation.
No bundler, no compile step, no virtual DOM.
Works in single-file demos or larger apps.
‚ÄúJust HTML‚Äù mental model ‚Äî easy to onboard teammates.

API & cheatsheet

Most apps only need a few primitives. Everything lives on x (and the extra alias). You can skim this later when you‚Äôre building more complex pages.

Store & reactivity

  • x.store ‚Äì reactive global state (proxied, saved in sessionStorage).
  • x.watch(path, fn) ‚Äì run fn(newVal, oldVal) when a path changes.
  • x.computed(name, fn) ‚Äì define derived state that tracks dependencies.

Templating & directives

  • (​(path)​) ‚Äì render store/computed values in text/attributes.
  • x-js="code" ‚Äì run inline JS when an element appears.
  • x-if, x-else, x-show, x-bind, x-on, x-for.

Config

extra.js exposes a reserved configuration object at x.config so future options can live in one place. Right now there are no official flags ‚Äî you can safely use it as a namespaced object for your own settings.

Example:

// Custom, app-level flags
x.config.appName = "My extra.js demo";
x.config.debug = true;
