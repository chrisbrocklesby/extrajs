What's included

Every feature below maps directly to the README. ExtraJS stays tiny while covering state, templating, directives, HTTP requests, and runtime helpers.

Core concepts

  • x.store - reactive global state (auto updates the DOM).
  • (​(path)​) - render store/computed values in text or attributes.
  • x-js - run inline JS when elements mount (async allowed).
  • x.computed - derived values that update automatically.
  • x.watch - run logic whenever a store path changes.
  • Session persistence â€“ x.store survives reloads via sessionStorage.
  • Manual re-scan â€“ x.apply() to process new DOM after swaps.

Directives

  • x-if / x-else - conditional insert/remove blocks.
  • x-show - toggle visibility with hidden (keeps DOM).
  • x-bind:attr - bind properties/attributes, including booleans.
  • x-on:event - inline event handlers with access to el.
  • x-for - loop over arrays with local variables.

HTTP helpers

  • x-http - fetch and swap HTML/text directly into targets.
  • x-method - override HTTP methods (GET/POST/DELETE/etc).
  • x-target / x-target-error - success and error targets.
  • x-swap - inner, append, prepend, before, after, outer.
  • x-trigger - click/submit, load, every:ms, event:ms debounce.
  • x-indicator - show loading elements while requests run.
  • x-headers - attach custom request headers.
  • x-confirm - confirm dialogs before sending.
  • x-form / x-json - form-encoded or JSON body helpers.
No bundler, no compile step, no virtual DOM.

API summary

Most apps only need a handful of primitives. Everything lives on x (and the extra alias).

Store & reactivity

  • x.store â€“ reactive global state (proxied, saved in sessionStorage).
  • x.watch(path, fn) â€“ run fn(newVal, oldVal) when a path changes.
  • x.computed(name, fn) â€“ define derived state that tracks dependencies.

Templating & directives

  • (​(path)​) â€“ render store or computed values in text/attributes.
  • x-js="code" â€“ run inline JS when an element appears.
  • x-if / x-else â€“ conditional insertion.
  • x-show â€“ toggle hidden without removing the element.
  • x-bind:attr â€“ bind a property/attribute to a store path.
  • x-on:event â€“ attach event handlers with full JS (sync/async).
  • x-for="item in items" â€“ simple loops with local variables.

Config

extra.js exposes a reserved configuration object at x.config so future options can live in one place. At the moment there are no official flags â€” it is safe to treat it as a simple namespaced object for your own settings.

Example (user-defined only):

CONFIG 
// Reserved for future built-in options
// You can store your own app-level flags here.
x.config.appName = "My extra.js demo";
x.config.debug = true;

When official config options are added, they will appear under x.config and be documented in the README.
